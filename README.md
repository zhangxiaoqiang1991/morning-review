# morning-review 晨间复盘

每天一次的结构化反思。AI 主动提问，帮你从工作日志中挖掘你没注意到的模式，提取今日优先级。

## 它用起来是什么感觉？

以下对话来自一次真实的晨间复盘（已脱敏）：

```
AI: 早上好。昨天你做了不少事——开发了新工具、写了复盘 skill、
    处理了一些飞书上的迁移工作。

    我观察到一个有意思的模式：你在情绪卡壳的时候，
    反而做出了当天最核心的产出。

    这让我想知道——**昨天有没有一个时刻，
    你发现原来自己想错了什么？**

你: 有的。我一直觉得情绪不好的时候什么都做不了，
    但昨天我发现情绪波动的时候，反而更适合做创造性的事。
    做工具、写东西的时候很容易进入心流，
    做完之后状态反而回来了。

AI: 所以你不是"等状态好了再行动"，
    而是"用行动修复状态"？

你: 对，就是这样。我从来没这么想过。

AI: 这个模式有名字——我叫它"修复型创作者"。
    大多数人的顺序：状态好 → 产出好
    你的顺序：状态不好 → 创造 → 状态变好。
    这个认知一旦被命名，你就可以主动用它了。

    ...

AI: 好。那今天你的 Top 3 优先级是什么？

你: 第一，优化复盘 skill 并录个使用视频。
    第二，消化一个关于思辨能力的框架。
    第三，处理昨天剩下的迁移工作。

AI: 胜出条件是什么？

你: 如果今天把那套思辨框架消化并形成了自己的理解，
    这一天就是值的。

AI: 我要说的是——**你的最佳产出往往诞生于你最不在状态的时候。
    情绪不是你的敌人，它是创造力的前置条件。**
    今天如果情绪有小波动，别压它，直接去做需要创造力的事。
```

## 设计思路：解决什么问题

### 1. 问题：复盘容易变成自我检讨

大多数复盘让你"填模板"——今天做得好、做得不好、明天干啥。但真正做不下去的原因不是没有模板，而是**没有人戳破你的含糊其辞**。

这个 skill 的做法：AI 主动从你的工作日志里读出你自己没意识到的模式，然后追问你、挑战你、连接你的经历到更大的框架。你不是在填空，是在和一个有判断力的对话者聊天。

### 2. 问题：复盘太深会变成内耗

反思到一定程度，容易陷入"我怎么哪哪都不对"的感觉。越反思越焦虑，反而影响行动。

这个 skill 的做法：

- **幸福瞬间（Phase 2.5）** — 四问都在"找问题"，所以专门加了一问"昨天有没有一个好的瞬间？"。铭记当下的幸福，提升幸福感，不是强行正能量，是防止过度反思。

  灵感来自稻盛和夫："活着，就要感谢。"

- **放下与出口（Phase 4.5）** — 复盘的最后一步不是保存，是放下。"拿到种子，放下烦恼。今天的三件事已经定了，开始今天。"

  灵感来自稻盛和夫："不要有感性的烦恼。反省完就放下，把精力投进新工作。"

### 3. 问题：只复盘行为不复盘念头

大多数复盘只在行为层面问"我做错了什么"，但曾国藩写日记三十年的核心方法是**"研几"**——不在结果处复盘，在念头处复盘。

你今天做出的一个让你后悔的决定，它的根源可能是一周前的一个你没注意到的念头。好的复盘不是改结果，是追溯念头。

这个 skill 的做法：在 Q1 认知反转中，如果你提到一个重要判断错误，AI 会追问——

> "在你做出那个决定之前，脑子里闪过的第一个念头是什么？"
> "那个念头是怎么发展成行动的？中间有没有一个时刻你觉得有点不对，但忽略过去了？"

你能追溯到决策之前的那个微妙念头，下一次同样念头出现时，就能提前识别。

### 4. 问题：复盘和行动脱节

复盘很好，但复完盘该干嘛？如果复盘不改变今天的行动，它就只是一种智力活动。

这个 skill 的做法：
- Phase 3 提取优先级 + 认知种子
- Phase 3.5 沉淀成 TODO + 一句话承诺
- 然后用"放下与出口"收束——**反思完了，投入新工作。**

## 数据源架构

复盘质量取决于输入数据的丰富度。核心是 Agent 工作日志（必需），辅助数据源用来覆盖更多工作轨迹。

> 原则：**有则用，无则跳过。** 辅助维度非必需，只靠核心日志也能完成复盘。

```
Agent 工作日志（90% 核心）
  ├── 飞书日历（日程结构）
  ├── Get 笔记（语音灵感）
  ├── Screenpipe（屏幕活动）
  └── 微信读书（知识输入）
```

### 辅助数据源安装

```bash
# 飞书 CLI
brew install lark-cli && lark-cli auth login

# Get 笔记 CLI
npm install -g @getnote/cli && getnote auth login

# Screenpipe（屏幕记录）
brew install screenpipe

# 微信读书 CLI
npm install -g weread-agent-cli
# 配置：weread config set-key "你的 API Key"
```
