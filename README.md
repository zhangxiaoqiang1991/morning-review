# morning-review 晨间复盘

每天一次的结构化反思。AI 主动提问，帮你从工作日志中挖掘你没注意到的模式，提取今日优先级。

## 它用起来是什么感觉？

```
AI: 早上好。回顾一下昨天——你在项目A上花了大量时间做迁移，
    同时穿插着写了两篇内容。但核心产出其实是那两篇文章。

    我观察到一个有意思的模式：你同时在做需要不同类型心智的事。
    一种是执行性的（迁移），一种是创造性的（写作）。
    你一直在两者之间来回切换。

    这让我想问：**昨天有没有一个时刻，你发现原来自己想错了什么？**

你: 有的。我以为迁移必须一次做完，但其实分几天做也完全可以，
    而且效率更高。

AI: 这个反转是什么触发的？
    ...

    （追问 2-3 轮后）

AI: 所以今天的 Top 3 优先级是：
    1. 把昨天那篇内容收尾发布
    2. 拆解一个你感兴趣的新方向
    3. 项目A只处理最紧急的部分，不急的放明天

    胜出条件：如果那篇内容今天发出去了，今天就是值的。
    今日避坑：不要在项目A上花超过 1 小时。
```

## 快速开始

```bash
git clone git@github.com:zhangxiaoqiang1991/morning-review.git
cp -r morning-review ~/.claude/skills/morning-review/
```

然后在 Claude Code 对话框中输入 `/morning-review` 启动。第一次运行会引导你完成配置。

## 四问复盘

| 问题 | 问什么 |
|------|--------|
| **认知反转** | 昨天有什么观点被颠覆了？ |
| **决策复盘** | 哪个决定最值得重看？ |
| **盲区发现** | 昨天忽略了什么？ |
| **能量信号** | 能量状态怎样？什么涨什么耗？ |

## 六维洞察透镜

AI 会从 6 个角度观察你的工作日志，找到你没注意到的模式：

| 透镜 | 它会看什么 |
|------|-----------|
| 模式显微镜 | 这个行为重复 10 次会形成什么？ |
| 决策基因解码 | 这个决策暴露了什么默认假设？ |
| 能量-产出关联 | 什么状态下产出最高？ |
| 身份张力诊断 | 你昨天切换了几个不同身份？ |
| 认知偏差命名 | 这个行为在心理学里叫什么？ |
| 反事实推演 | 如果做了不同选择会怎样？ |

## 完整工作流

```
Phase 0  加载工作日志
Phase 1  昨日速览（2 分钟）
Phase 1.5  AI 观察 + 六维洞察（3 分钟）
Phase 2  四问互动复盘（15 分钟）
Phase 3  认知种子 + 优先级提取（5 分钟）
Phase 4  误判心理学对照（可选）
Phase 5  保存并打开
```

## 数据源架构

复盘质量取决于输入数据的丰富度。核心是 Agent 工作日志（必需），辅助数据源用来覆盖更多工作轨迹。

```
Agent 工作日志（90% 核心）
  ├── 飞书日历（日程结构）
  ├── Get 笔记（语音灵感）
  ├── Screenpipe（屏幕活动）
  └── 微信读书（知识输入）
```

> 原则：**有则用，无则跳过。** 辅助维度非必需，只靠核心日志也能完成复盘。

### 辅助数据源安装

```bash
# 飞书 CLI
brew install lark-cli && lark-cli auth login

# Get 笔记 CLI
npm install -g @getnote/cli && getnote auth login

# Screenpipe（屏幕记录）
brew install screenpipe

# 微信读书 CLI
npm install -g weread-agent-cli
# 配置：weread config set-key "你的 API Key"
```

## 底层逻辑

大多数复盘工具让你"填模板"，但复盘做不下去的原因不是没有模板，而是没有人戳破你的含糊其辞。

这个 skill 的核心差异：
- **洞察先行** — AI 从工作日志中读出你没注意到的模式
- **六维透镜** — 6 个观察维度拆解你的一天
- **三味燃料** — 每一层洞察都注入误判心理学（Munger 25 种）、跨学科思维模型、历史典故。帮你看到更大的坐标系
- **认知种子** — 每次复盘产出一颗可迁移的洞察
