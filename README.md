# morning-review 晨间复盘

一个帮你做深度复盘的 AI 教练。每天 15 分钟，基于你的工作日志，AI 主动追问、捕捉你没注意到的模式，提取今日优先级。

它主要解决三个问题：一个人复盘深度不够、反思过度变成内耗、复完盘还是不知道今天该干啥。

## 它用起来是什么感觉？

以下对话来自一次真实的晨间复盘（已脱敏）：

```
AI: 早上好。昨天你做了不少事——开发了新工具、写了复盘 skill、
    处理了一些飞书上的迁移工作。

    我观察到一个有意思的模式：你在情绪卡壳的时候，
    反而做出了当天最核心的产出。

    这让我想知道——**昨天有没有一个时刻，
    你发现原来自己想错了什么？**

你: 有的。我一直觉得情绪不好的时候什么都做不了，
    但昨天我发现情绪波动的时候，反而更适合做创造性的事。
    做工具、写东西的时候很容易进入心流，
    做完之后状态反而回来了。

AI: 所以你不是"等状态好了再行动"，
    而是"用行动修复状态"？

你: 对，就是这样。我从来没这么想过。

AI: 这个模式有名字——我叫它"修复型创作者"。
    大多数人的顺序：状态好 → 产出好
    你的顺序：状态不好 → 创造 → 状态变好。
    这个认知一旦被命名，你就可以主动用它了。

    ...

AI: 好。那今天你的 Top 3 优先级是什么？

你: 第一，优化复盘 skill 并录个使用视频。
    第二，消化一个关于思辨能力的框架。
    第三，处理昨天剩下的迁移工作。

AI: 今日标准是什么？

你: 如果今天把那套思辨框架消化并形成了自己的理解，
    这一天就是值的。

AI: 我要说的是——**你的最佳产出往往诞生于你最不在状态的时候。
    情绪不是你的敌人，它是创造力的前置条件。**
    今天如果情绪有小波动，别压它，直接去做需要创造力的事。
```

## 它能解决你的什么问题？

它主要解决三个问题：一个人复盘深度不够、反思过度变成内耗、复完盘还是不知道今天该干啥。

**问题一：一个人复盘，深度不够。**
你坐在那儿硬想，大脑会本能地绕开真正费劲的问题。写出来的永远是安全的那一层——做了啥、没做啥、明天干啥。更深层的东西（行为背后的念头、决策时的犹豫、一闪而过的直觉）全被跳过了。

→ 让 AI 当你的复盘教练。它读你的日志，主动追问、在你含糊时戳破你。借鉴曾国藩的溯念头法：不止在行为层面复盘，追到念头层面——"做出那个决定之前，脑子里闪过的第一个念头是什么？"

**问题二：反思到深处，变成内耗。**
从数据不好追到"我是不是不适合干这个"，从决策失误追到"我能力不行"。这不是复盘，是自我审判。

→ 专门设计了两个出口。幸福瞬间：问一句"昨天有没有一个好的瞬间"，防止过度反思。放下与出口：复盘结束时说"拿到种子，放下烦恼，去做事"。思路来自稻盛和夫：反省完就放下，不内耗。

**问题三：复完盘，还是不知道今天该干啥。**
复盘如果不变今天的行动，就是智力活动。

→ 复完盘必须产出三样东西：一颗可迁移的认知种子、三个优先事项、一个可执行的 TODO。复完直接开工。

### 设计考虑

三个灵感来源在问题中已分别说明，这里补充两个设计原则：

- **发芽机制（借鉴 Get 笔记）：** 每次复盘产出一颗可跨域迁移的认知种子，以后在其他场景自动发芽
- **数据源有则用无则跳过：** Agent 工作日志占 90%，辅助接入飞书日历、Get 笔记、Screenpipe、微信读书，但不强制

## 数据源架构

复盘质量取决于输入数据的丰富度。核心是 Agent 工作日志（必需），辅助数据源用来覆盖更多工作轨迹。

> 原则：**有则用，无则跳过。** 辅助维度非必需，只靠核心日志也能完成复盘。

```
Agent 工作日志（90% 核心）
  ├── 飞书日历（日程结构）
  ├── Get 笔记（语音灵感）
  ├── Screenpipe（屏幕活动）
  └── 微信读书（知识输入）
```

### 辅助数据源安装

```bash
# 飞书 CLI
brew install lark-cli && lark-cli auth login

# Get 笔记 CLI
npm install -g @getnote/cli && getnote auth login

# Screenpipe（屏幕记录）
brew install screenpipe

# 微信读书 CLI
npm install -g weread-agent-cli
# 配置：weread config set-key "你的 API Key"
```
