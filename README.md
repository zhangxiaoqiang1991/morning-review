# morning-review 晨间复盘

每天一次的结构化反思。基于工作日志，通过 4 轮追问驱动认知复盘 + 今日优先级提取。

## 核心理念

**复盘不是写日记，是主动审视自己的思考过程。**

工作日志记录了"你做了什么"，晨间复盘追问"你为什么那么做"。
日志是客观记录，晨间复盘是主观审视。两者缺一不可。

## 底层逻辑

大多数复盘工具让你"填模板"，但复盘做不下去的原因不是没有模板，而是没有人戳破你的含糊其辞。

这个 skill 的核心差异：
- **洞察先行** — 不只问问题，而是从你的工作日志中读出你没注意到的模式
- **六维透镜** — 从 6 个不同的观察维度拆解你的一天（模式显微镜、决策基因解码、能量关联、身份张力、认知偏差、反事实推演）
- **三味燃料** — 每一层洞察都注入误判心理学（Munger 25 种）、跨学科思维模型（20 个）、或全球历史典故（30+），让你看到更大的坐标系
- **认知种子** — 每次复盘产出一颗可迁移的认知点，脱离今天场景依然成立

## 数据源架构：尽可能覆盖你的工作轨迹

复盘的质量取决于输入数据的丰富度。核心设计思路是**多维覆盖**：

```
Agent 工作日志（90% 核心）
  ├── 飞书日历/文档（日程结构与文档动态）
  ├── Get 笔记（语音灵感输入）
  ├── Screenpipe（屏幕活动概况）
  └── 微信读书（知识输入）
```

最核心的是 **Agent 工作日志**（如 Claude Code 日报），它反映了你的主要工作产出。其余数据源作为辅助交叉参考，用来补充 Agent 可能覆盖不到的维度（如会议时间、灵感记录、跨应用活动、阅读输入）。

> 原则：**有则用，无则跳过。** 后四个维度是辅助而非必需，不具备条件时只靠核心日志也能完成复盘。

## 工作流程

```
Phase 0  加载工作日志（多维数据源融合）
Phase 1  昨日速览（2 分钟）
Phase 1.5  AI 观察 + 六维透镜洞察（3 分钟）
Phase 2  四问互动复盘（15 分钟）
Phase 3  认知种子 + 优先级提取（5 分钟）
Phase 4  误判心理学对照（可选，3 分钟）
Phase 5  保存到笔记并打开
```

## 四问复盘

1. **认知反转** — 昨天有什么观点被颠覆了？
2. **决策复盘** — 哪个决定最值得重看？
3. **盲区发现** — 昨天忽略了什么？
4. **能量信号** — 能量状态怎样？什么涨什么耗？

## 六维洞察透镜

| 透镜 | 核心问题 |
|------|---------|
| 模式显微镜 | 这个行为重复 10 次会形成什么？ |
| 决策基因解码 | 这个决策暴露了什么默认假设？ |
| 能量-产出关联 | 什么状态下什么产出最高？ |
| 身份张力诊断 | 今天同时穿了几件衣服？ |
| 认知偏差命名 | 这个行为在心理学里叫什么？ |
| 反事实推演 | 如果做了不同选择会怎样？ |

## 快速模式

赶时间时输入 `/morning-review quick`，5 分钟完成核心复盘 + 优先级提取。

## 安装

### 前置条件
- Claude Code / Codex / OpenClaw / KimiCode / Trae / Workbuddy 等 AI 编程工具
- （可选）工作日志系统，用于提供昨日数据

### 步骤

```bash
# 1. 将目录放到 Claude Code skills 目录
cp -r morning-review ~/.claude/skills/

# 2. 在对话中输 /morning-review 启动
```

### 核心原则：尽可能覆盖你的工作轨迹

复盘质量取决于输入数据的丰富度。核心是 Agent 工作日志（必需），其余辅助数据源用来覆盖更多工作轨迹，**有则用，无则跳过**。

### 辅助数据源安装命令

```bash
# 飞书 CLI（日历/文档）
# 安装：brew install lark-cli
lark-cli calendar +agenda

# Get 笔记 CLI（语音灵感记录）
# 安装：npm install -g @getnote/cli
getnote notes --all --limit 5

# Screenpipe（屏幕活动记录）
# 安装：brew install screenpipe
curl "http://localhost:3030/search?content_type=ocr&limit=10"

# 微信读书 CLI（阅读记录）
# 安装：npm install -g weread-agent-cli
weread shelf recent --json --limit 3
```

企业微信、钉钉等办公平台同样可以用 CLI/API 拉取日程和文档。原则：**能覆盖就覆盖，不需要全都配齐。**
